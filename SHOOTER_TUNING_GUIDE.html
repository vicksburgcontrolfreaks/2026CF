1. Understanding the Physics

Figure 1: Trajectory comparison. At 62°, you achieve MORE range with the SAME velocity compared to 68°. To hit the same target, you need LESS RPM.

Physics Background

For projectile motion with launch angle θ and initial velocity v:

Horizontal velocity: v_x = v × cos(θ) Vertical velocity: v_y = v × sin(θ) Range (distance): R = v² × sin(2θ) / g At 68°: sin(136°) ≈ 0.695 At 62°: sin(124°) ≈ 0.829 Result: 62° has ~19% BETTER range efficiency than 68°!

What this means for tuning: To hit the same target at 62° that you used to hit at 68°, you need approximately 8-12% LESS wheel RPM. This is a favorable change that should improve consistency and reduce motor stress.

// File: src/main/java/frc/robot/constants/ShooterConstants.java // OLD velocity table (68° launch angle - DELETE THIS): public static final double[][] kShooterVelocityTable = { {1.75, 2950}, {2.69, 3450}, {3.3, 4000}, {4.8, 4500}, }; // NEW velocity table (62° launch angle - expect ~8-12% lower RPM): public static final double[][] kShooterVelocityTable = { {1.75, 2650}, // Update with YOUR test result for ~1.75m {2.69, 3100}, // Update with YOUR test result for ~2.69m {3.3, 3600}, // Update with YOUR test result for ~3.3m {4.8, 4050}, // Update with YOUR test result for ~4.8m // Add more entries if you tested additional distances };

⚠️ Important: Use Vision Distance, Not Measured Distance

When creating the final table, use the vision-reported distance (from Shooter/Distance to Target), NOT your tape-measured distance. Why?

  • The robot uses vision distance for real-time RPM calculation during matches
  • Vision distance may differ from tape-measured due to camera mounting errors (angular offset, lens-to-sensor offset)
  • Using vision distance ensures the table matches what the robot "sees" during competition

Example: If you measured 3.0m with tape but vision reports 3.15m, use 3.15m in your table entry.

9. Tips for Efficient Tuning

Best Practices

  • Start close, work outward: Tune 1.75m first (easiest), then progressively longer distances
  • Binary search for RPM: Make large adjustments (±400) initially, then refine with smaller steps (±100)
  • Expect lower RPM: Your starting guess should be 8-12% below old values due to better range efficiency at 62°
  • Consistent test conditions: Use same battery charge level (>12.0V), same game pieces, same field conditions
  • 3-shot confirmation: Don't finalize an RPM until 3 consecutive shots score successfully
  • Document everything: Take notes on what worked and what didn't - helps with future tuning
  • Test moving shots separately: Velocity compensation may need separate tuning
  • Have spare game pieces ready: Tuning can take 20-50 test shots, prepare accordingly
  • Two-person team: One person drives/operates, one person adjusts RPM and records data
  • High-speed video: Film a few shots in slow-motion to verify trajectory looks correct for 62°
